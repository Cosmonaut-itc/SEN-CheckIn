Fundamentos

Guía de Diseño

Este documento define los tokens, componentes y lineamientos visuales para la plataforma web de SEN con la Paleta Michoacán. Para iOS y Android, consulta el Design System Mobile. Cada decisión de diseño tiene una razón. Usa esta guía como referencia para construir interfaces web que se sientan SEN.

✓ Principio
Todos los productos de SEN comparten la misma estructura de tokens. Jale, Bodega, Caja y Ruta se sienten familia — pero cada uno tiene su propia paleta regional.
✗ Evita
Nunca inventes colores, tamaños o radios de borde fuera de este sistema. Si necesitas algo nuevo, proponlo al equipo de diseño.

Guías

Lineamientos

Reglas para mantener la consistencia entre todos los productos de SEN.

Botones

✓ Así sí
Un solo botón primario (cobre michoacano) por pantalla. Es la acción principal que quieres que el usuario tome.
✗ Así no
Múltiples botones cobre compitiendo por atención. Si todo es importante, nada es importante.

Colores

✓ Así sí
Usa los tokens CSS. var(--accent-primary) se adapta automáticamente entre light y dark.
✗ Así no
Colores hex directos como #B8602A. Si cambiamos un color, todo lo hardcodeado se rompe.

Espaciado

✓ Así sí
Espacio generoso entre secciones (space-16 o más). El contenido respira. Las PyMEs usan esto en pantallas chicas — necesitan espacio.
✗ Así no
Todo apretado para "aprovechar la pantalla". Eso genera fatiga visual y errores de clic en móvil.

Texto

✓ Así sí
"Tu nómina se procesó." Directo, en segunda persona, sin jerga.
✗ Así no
"El proceso de cálculo de remuneraciones ha sido ejecutado satisfactoriamente en el sistema."
Guías

Accesibilidad

Nuestros usuarios no son developers. Muchos trabajan desde celulares viejos con pantallas chicas. La accesibilidad es una obligación, no un extra.

Contraste mínimo: 4.5:1 para texto body, 3:1 para texto grande (WCAG AA)
Touch targets: Mínimo 44x44px en móvil. El personal de piso checa desde su celular con las manos ocupadas.
Tamaño mínimo: 16px para cuerpo de texto en pantalla. Nunca menos de 12px para nada.
Color no es el único indicador: Siempre combina color con texto o ícono. "Retardo" no solo en naranja, también con la etiqueta.
Focus visible: Todos los elementos interactivos tienen un estado de focus claro con el ring cobre.
Etiquetas: Todos los inputs tienen label visible. Nada de solo placeholder como etiqueta.
Guías

Movimiento

Las animaciones en SEN son sutiles y funcionales. Comunican que algo pasó, no son decoración.

CURVAS DE ANIMACIÓN
--transition-fast · 150ms · Hovers, toggles, micro-interacciones
--transition-base · 250ms · Transiciones de página, modales, paneles
--transition-slow · 400ms · Animaciones de entrada, page transitions
✓ Así sí
Transiciones suaves que confirman acciones: un toast aparece suavemente, una fila se desvanece al eliminarla, un modal entra con fade + scale sutil.
✗ Así no
Bouncing, rebotes exagerados, animaciones lentas que bloquean la interacción, efectos de parallax que marean. Esto es software de trabajo, no un portfolio.
