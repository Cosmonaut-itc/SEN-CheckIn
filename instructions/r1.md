# Release 01 – Monorepo base + paquetes compartidos (incl. contrato Eden)

## Rol de esta release

Dejar listo el esqueleto del monorepo:

- bun + Turborepo.
- Carpetas `apps/` y `packages/`.
- Paquetes compartidos para:
    - configuración (`@sen-checkin/typescript-config`),
    - tipos de dominio (`@sen-checkin/types`),
    - configuración (`@sen-checkin/eslint-config`),
    - contrato API Elysia/Eden (`@sen-checkin/api-contract`).
- Carpeta `documentacion/` con los primeros archivos de docs por release.

No se implementa lógica de negocio ni endpoints reales todavía.

## Paso 3 – Crear paquetes compartidos (`packages/`)

### 3.1 Paquete `@sen-checkin/types`

Carpeta: `packages/types`.

- `package.json` con:
    - Nombre de paquete interno, por ejemplo `"@asistencia/types"`.
- Carpeta `src/` con un archivo principal, p.ej. `index.ts`:
    - Incluir solo definiciones muy simples o incluso solo comentarios:
        - Comentario indicando que aquí se declararán tipos de dominio:
            - `Employee`
            - `Device`
            - `AttendanceRecord`
            - etc.
        - No definir todavía la estructura detallada, solo bocetos a nivel de comentario.

Este paquete será la **fuente de verdad** para tipos de dominio compartidos entre backend y frontend.

### 3.3 Paquete `@sen-checkin/api-contract` (Elysia + Eden)

Carpeta: `packages/api-contract`.
Referencia: https://elysiajs.com/eden/overview.html
Revisa la referencia antes de crear el paquete. Este paquete tiene que estar funcional, ya esta una app base de Elysia, con eso puedes crear el paquete.

- `package.json` con:
    - Nombre `"@sen-checkin/api-contract"`.
    - Este paquete será consumido tanto por `apps/api` como por `apps/mobile`.

- Carpeta `src/` con un archivo principal (por ejemplo `index.ts`) que contenga:
    - Comentarios describiendo el propósito:
        - Aquí se publicará el tipo del servidor Elysia.
        - Aquí se expondrán factories para crear clientes Eden tipados en el frontend.
    - Implementar la lógica necesaria para que el paquete funcione correctamente.
    - Revisar la referencia antes de crear el paquete, prueba que pueda ser exportado y consumido correctamente en la app de mobile.
